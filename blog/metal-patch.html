Patching Metal Graphics into vphone

Feb 2026 / wh1te4ever/super-tart-vphone

vphone's paravirtualized GPU has no Metal because the shader compiler plugin is trapped in the PCC dyld shared cache — extracted images have no relocations and can't load. Clean-room reimplementation from RE, ~230 lines of C++.

AI-assisted install

Save main.mm and build.sh to CFW/libAppleParavirtCompilerPluginIOGPUFamily/, then give this prompt to Claude Code, Codex, or similar:

The vphone VM needs the Metal compiler plugin built and installed. Source is at CFW/libAppleParavirtCompilerPluginIOGPUFamily/main.mm, build script at build.sh. Do the following:
  1. Run ./build.sh to compile the dylib.
  2. Boot DFU: ./.build/debug/tart run vphone --dfu &
  3. Boot SSH ramdisk: cd CFW && ./boot_rd.sh then iproxy 2222 22 &
  4. SSH in (root@127.0.0.1 -p2222, pw alpine), mount rootfs: /sbin/mount_apfs -o rw /dev/disk1s1 /mnt1
  5. SCP dylib to /mnt1/System/Library/Extensions/AppleParavirtGPUMetalIOGPUFamily.bundle/
  6. Halt: sync && /sbin/halt
  7. Kill tart, normal boot: ./.build/debug/tart run vphone --vnc-experimental
The rest of the bundle is already on rootfs from fix_boot.py.

cd CFW/libAppleParavirtCompilerPluginIOGPUFamily

Verify

  • backboardd running (no crash)
  • SpringBoard renders with a background, not black
  • MetalTest: Metal Device Create Success: Apple Paravirtual device

Why this is necessary

The VM's paravirt GPU kernel driver works, but Metal needs a userspace plugin at /System/Library/Extensions/AppleParavirtGPUMetalIOGPUFamily.bundle. The main binary, plist, and codesig copy from the PCC disk image. The compiler plugin only exists in the PCC dyld shared cache — extracted Mach-Os have cache-relative vmaddrs, no LC_DYLD_CHAINED_FIXUPS, no rebase/bind opcodes. dyld can't load them. backboardd crashes, black screen.

What the plugin does

Four exports: MTLCompilerCreate (version==8 check, stores target triple), MTLCompilerBuildRequestWithOptions (AIR module in, metallib out), MTLCompilerDelete, MTLCompilerReleaseReply. Calls libGPUCompiler and libLLVM from the device DSC. Key flags: -undefined dynamic_lookup (runtime symbol resolution) and -not_for_dyld_shared_cache.

RE notes

From the PCC DSC. 47 functions, mostly C++ thunks. AppleParavirtCompiler class with vtable + 3 members. arm64e auth_stubs confirmed the import set. Entry point search walks air.vertex/air.fragment/air.kernel metadata nodes. Original used C++ LLVM API; reimplementation uses the stable C API (exported since LLVM 3.x).
