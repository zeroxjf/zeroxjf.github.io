Reimplementing Apple's Paravirt GPU Compiler Plugin
The VM's paravirtualized GPU needs a Metal compiler plugin that only exists inside the PCC dyld shared cache. You can't extract it — DSC images have no relocations. Reverse-engineered and rewrote it.
Feb 2026

1. Save the source files

Save both to CFW/libAppleParavirtCompilerPluginIOGPUFamily/:

  • main.mm — reimplemented compiler plugin (~230 lines)
  • build.sh — compiles + ad-hoc signs the dylib

2. AI-assisted build & install

Give this prompt to Claude Code, Codex, or similar:

The vphone VM needs the Metal compiler plugin built and installed. The source is at CFW/libAppleParavirtCompilerPluginIOGPUFamily/main.mm and the build script is CFW/libAppleParavirtCompilerPluginIOGPUFamily/build.sh. Do the following:
  1. Run ./build.sh in CFW/libAppleParavirtCompilerPluginIOGPUFamily/ to compile the dylib.
  2. Boot the VM in DFU mode: ./.build/debug/tart run vphone --dfu &
  3. Boot the SSH ramdisk: cd CFW && ./boot_rd.sh then iproxy 2222 22 &
  4. SSH into the ramdisk (root@127.0.0.1 -p2222, password alpine), mount the rootfs writable: /sbin/mount_apfs -o rw /dev/disk1s1 /mnt1
  5. SCP the built dylib to /mnt1/System/Library/Extensions/AppleParavirtGPUMetalIOGPUFamily.bundle/ on the device.
  6. Halt the device: sync && /sbin/halt
  7. Kill tart, then normal boot: ./.build/debug/tart run vphone --vnc-experimental
The rest of the bundle (main binary, Info.plist, CodeResources) is already on the rootfs from fix_boot.py. You're only replacing the compiler plugin dylib.

3. Manual build & install

# Build
cd CFW/libAppleParavirtCompilerPluginIOGPUFamily
chmod +x build.sh && ./build.sh

# Boot DFU + ramdisk
./.build/debug/tart run vphone --dfu &
cd CFW && ./boot_rd.sh
iproxy 2222 22 &

# Mount rootfs, copy dylib
ssh root@127.0.0.1 -p2222 "/sbin/mount_apfs -o rw /dev/disk1s1 /mnt1"
scp -P 2222 libAppleParavirtCompilerPluginIOGPUFamily.dylib \
    root@127.0.0.1:/mnt1/System/Library/Extensions/AppleParavirtGPUMetalIOGPUFamily.bundle/

# Halt + normal boot
ssh root@127.0.0.1 -p2222 "sync && /sbin/halt"
# wait for VM to stop, kill tart
./.build/debug/tart run vphone --vnc-experimental

SSH password is alpine.

4. Verify

  • backboardd running (no crash)
  • SpringBoard renders with a background, not a black screen
  • MetalTest prints Metal Device Create Success: Apple Paravirtual device

Why this is necessary

The VM uses a paravirtualized GPU (AppleParavirtGPU). The kernel sees it, but Metal needs a userspace plugin bundle at /System/Library/Extensions/AppleParavirtGPUMetalIOGPUFamily.bundle. The main binary, Info.plist, and CodeResources copy from the PCC disk image. The compiler plugin dylib is the problem — it only exists inside the PCC dyld shared cache.

Extracting it doesn't work. The resulting Mach-O has a cache-relative vmaddr, no LC_DYLD_INFO_ONLY/LC_DYLD_CHAINED_FIXUPS, no rebase/bind opcodes. dyld can't relocate it. backboardd tries to dlopen it, fails, CoreAnimation crashes in CA::OGL::MetalContext::get_tile_pipeline. Black screen.

What the plugin does

Metal shader compiler for the paravirt GPU. Four exports: MTLCompilerCreate (checks version==8, stores target triple), MTLCompilerBuildRequestWithOptions (takes LLVM IR / AIR module, compiles to metallib), MTLCompilerDelete, MTLCompilerReleaseReply. Calls into libGPUCompiler.dylib and libLLVM.dylib in the device DSC — both already loaded by Metal.framework at runtime.

Key linker flags

  • -undefined dynamic_lookup — defers LLVM/GPUCompiler symbol resolution to runtime
  • -not_for_dyld_shared_cache — required with dynamic_lookup
  • -install_name ... — must match the on-device load path

RE notes

Reverse-engineered from the PCC DSC (System/Library/Caches/com.apple.dyld/dyld_shared_cache_arm64e). 47 functions, mostly C++ thunks. Class AppleParavirtCompiler with vtable, three members. arm64e auth_stubs confirmed the exact import set. Entry point search iterates air.vertex/air.fragment/air.kernel metadata nodes. Original used C++ LLVM API; reimplementation uses the stable C API (exported since LLVM 3.x).
